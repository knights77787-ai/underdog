# 최종 점검 체크리스트

## 0) 배포 전 기본
- [ ] 백엔드 재시작 완료 (`alembic` 적용된 DB 기준)
- [ ] 브라우저 강력 새로고침(캐시 제거) 완료
- [ ] `.env` 값이 실제 프로세스에 반영됐는지 확인 (`custom_match_debug`의 `thr`, `rms` 로그)

## 1) 공통 기능/기동
- [ ] `GET /health` 응답 확인: `ok`, `stt_enabled` 정상
- [ ] 메인 페이지 접속 시 콘솔 치명 에러 없음
- [ ] WebSocket 연결 성공 (`/ws` 업그레이드 정상)

## 2) 소리 등록/목록/재생
- [ ] `new-sound`에서 파일 업로드 등록 성공
- [ ] 등록 직후 목록 탭에 즉시 표시
- [ ] 재생 버튼 클릭 시 오디오 재생 성공 (404/410 처리 정상)
- [ ] 중복 이름 등록 시 차단/안내 토스트 정상

## 3) 세션/로그인 유지
- [ ] 로그인 상태에서 등록한 소리가 로그아웃 후 재로그인해도 목록에 유지
- [ ] 게스트 세션에서는 기존처럼 세션 기준으로만 보이는지 확인
- [ ] 로그아웃 시 세션 정리 API 호출되고 홈으로 이동

## 4) 커스텀 인식 정확도
- [ ] 등록 소리 재생 시 커스텀 감지 발생 (`custom_match` 로그 확인)
- [ ] 무음/일반 대화만 있을 때 커스텀 오탐이 줄었는지 확인
- [ ] 유사 소리 여러 개 등록 시 특정 라벨로만 고정되지 않고 합리적으로 판정되는지 확인
- [ ] 같은 소리 반복 시 쿨다운 간격 내 중복 알림 억제 확인

## 5) 알림/UI 일관성
- [ ] live/new_sound/settings 토스트 스타일(위험/성공/경고) 일관성 확인
- [ ] 상태줄 메시지와 토스트 메시지 내용이 충돌 없이 자연스러운지 확인
- [ ] 가이드 모달 “다음에 보지 않기” 체크 동작 확인

## 6) 운영 관측(로그/DB)
- [ ] `events`에 아래 컬럼 저장 확인:
  - [ ] `matched_custom_sound_id`
  - [ ] `custom_similarity`
  - [ ] `custom_threshold_used`
  - [ ] `custom_rms`
  - [ ] `custom_pick_reason`
- [ ] `custom_match_debug` 로그가 주기적으로 출력되어 튜닝 가능 상태인지 확인

## 7) 모바일(안드로이드) 진동
- [ ] 첫 터치 후 알림 발생 시 진동 동작 확인
- [ ] 미동작 시 기기 진동/브라우저 권한/HTTPS 확인
- [ ] 백그라운드 진동은 웹 한계(FCM 자동 푸시 미연결) 인지 확인
