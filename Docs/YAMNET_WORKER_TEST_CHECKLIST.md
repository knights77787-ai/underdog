# YAMNet Worker Comparison Checklist (2 vs 3)

이 문서는 `YAMNet worker=2` 와 `YAMNet worker=3` 설정을 비교해,
실제 운영에 어떤 값이 더 적합한지 판단하기 위한 체크리스트입니다.

## 1) 사전 조건

- [ ] 동일한 서버/네트워크 환경에서 테스트한다.
- [ ] 동일한 오디오 샘플(사이렌, 클락션, 커스텀 소리)을 사용한다.
- [ ] 각 케이스 테스트 시간을 동일하게 맞춘다(권장: 5~10분).
- [ ] `ENABLE_ML_WORKERS=1` 상태를 확인한다.
- [ ] 비교군을 명확히 분리한다:
  - [ ] Case A: YAMNet workers = 2
  - [ ] Case B: YAMNet workers = 3

## 2) 시나리오 A - 비언어 단독 부하

- [ ] 사이렌 30초 재생 x 5회
- [ ] 클락션 10초 재생 x 10회
- [ ] 커스텀 소리 5초 재생 x 10회
- [ ] 각 이벤트가 누락 없이 들어오는지 기록
- [ ] 이벤트 발생까지 체감 지연(초) 기록

## 3) 시나리오 B - 혼합 부하(실사용 유사)

- [ ] 비언어 재생 중 마이크로 키워드 발화(`불`, `비상`) 반복
- [ ] `caption` / `alert` 이상 중복 여부 확인
- [ ] danger/caution 우선순위 표시 정상 여부 확인
- [ ] 프론트 자막/로그 반영 지연 기록

## 4) 수집 지표 (각 케이스 공통)

- [ ] `AUDIOCLS_QUEUE` 평균 길이
- [ ] `AUDIOCLS_QUEUE` 최대 길이
- [ ] YAMNet 처리 건수(초당)
- [ ] YAMNet 평균 처리 시간(ms)
- [ ] alert 누락 건수
- [ ] CPU 평균/최대 사용률
- [ ] 메모리 평균/최대 사용량
- [ ] 에러 로그 발생 횟수(예외/queue full 등)

## 5) 합격 기준(예시)

- [ ] 누락률 1% 이하
- [ ] alert 지연 p95 2초 이하
- [ ] 큐 최대치가 장시간 포화되지 않음
- [ ] CPU 최대치가 임계치(예: 85~90%)를 장시간 넘지 않음
- [ ] 메모리 급증/프로세스 재시작 없음

## 6) 최종 판단 가이드

### 3개 채택

- [ ] 큐 적체/지연이 유의미하게 감소
- [ ] CPU/RAM 증가가 허용 범위
- [ ] 안정성(오류/누락) 악화가 없음

### 2개 유지

- [ ] 3개 대비 성능 개선이 미미함
- [ ] CPU/RAM 부담이 크게 증가함
- [ ] 오히려 오류/지연이 증가함
